Badge Protection

The badge cannot be faked.

Three layers of protection make unauthorized badge use impossible — and immediately visible to anyone who clicks it.

⚡

Live API on every load

The badge calls our API every time the page loads. If the domain isn't registered, it renders grey and says UNVERIFIED. An unauthorized store gets a badge that exposes rather than protects them.

🔗

Domain-locked registry link

Every badge links to idrshield.com/verify/[domain]. If someone copies another store's badge code, clicking it shows the wrong store's registry — immediately exposing the fraud to any visitor who checks.

🖼

Screenshot-protected install

Wix and Webflow users install their badge as a screenshot from their live verify page, linked back to idrshield.com/verify/theirdomain. Anyone who clicks it sees the real registry record — misuse is immediately visible.

Usage Guidelines

The badge earns authority through consistent use.

Every misuse reduces the signal value for every enrolled store.

✓ Correct usage

✓Dark seal on white, cream, and light backgrounds
✓Gold outline on black, navy, and dark backgrounds
✓Always link the badge to your live verification page
✓Display your Founding Member badge proudly — it cannot be replicated
✓Keep it as a live embed so the ring pulse signals real-time monitoring

✗ What to avoid

✗Never place the badge without a working link to your registry record
✗Never use dark seal on dark backgrounds — use the gold outline instead
✗Never alter colours, typeface, or add extra shadows or glow effects
✗Never screenshot and upload as a static image — the animated ring will be missing
✗Never display Active status if your membership has lapsed — the embed handles this automatically
